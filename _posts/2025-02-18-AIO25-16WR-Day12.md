## The Knight's Tour problem

### Mô tả

**Bài toán Mã đi tuần** (hay Hành trình của quân mã) là bài toán về việc di chuyển một quân mã trên bàn cờ vua (8 x 8 ô). Quân mã được đặt ở một ô trên một bàn cờ trống, sau đó di chuyển theo quy tắc của cờ vua để đi qua mỗi ô trên bàn cờ **đúng một lần**.

Nếu một quân mã đi hết 64 vị trí trên bàn cờ, và tại vị trí cuối cùng có thể di chuyển đến vị trí bắt đầu thông qua 1 bước cờ, thì đó gọi là một hành trình đóng. Đối với những hành trình mà điểm đầu và điểm cuối không cách nhau trong 1 bước cờ, thì đso gọi là hành trình mở.

Ý tưởng: Xuất phát từ 1 ô, gọi số bước đi là $t = 1$

Ta cho quân mã thử đi tiếp 1 trong 8 ô có thể đi:
- nếu ô này chưa từng được mã đi qua thì chọn làm bước đi tiếp theo. 
- trong trường hợp không còn ô nào chưa đi, quay lại bước trước đó và thử đi hướng khác

Tại mỗi nước đi, kiểm tra xem tổng số nước đi bằng n*n chưa, 
- nếu bằng rồi thì mã đã đi qua tất cả các ô, dừng hàm (do chỉ cần tìm một giải pháp).
- nếu chưa bằng, gọi đệ quy để chọn nước đi tiếp theo

Ví dụ về bài toán Mã đi tuần với bàn cờ size 5x5

![Knight's Tour example 5x5](/my-blog/assets/img/20250218-AIO25-16WR-Day12/knight-tour-5x5.png)

Ý tưởng thuật toán
- **Input**: chessboard size n x n (n), starting position (start_x, start_y)
- **Output**: Find the sequence ò moves for the knight to víit every square exactly once

{% highlight python %}
procedure KNIGHTS_TOUR(x, y, move_count, board)
  if move_count = N**2 then
    return true                                                                     # Hoàn tất tìm được lời giải khi số bước bằng size bàn cờ n x n
  end if
  for each (dx, dy) in possible_moves do                                            # Kiểm tra các bước có thể đi của mã
    new_x <- x + dx
    new_y <- y + dy
    if (0 <= new_x < N) and (0 <= new_y < N) and (board[new_x][new_y] = -1) then    # Xét điểm mới nằm trong size bàn cờ (>=0 và < N) và chưa được đi qua (=-1)
      board[new_x][new_y] <- move_count                                             # Nếu thỏa điều kiện, note điểm này là bước thứ move_count
      if KNIGHTS_TOUR(new_x, new_y, move_count+1, board) then                       # Tiếp tục xét move_count+1
        return true
      end if
      board[new_x][new_y] <- -1                                                     # Nếu điểm mới không thỏa điều kiện, xét các điểm còn lại, đưa vị trí đang xét về thành "chưa đi qua" (-1), đây là thuật toán quay lui (backtracking)
    end if
  end for
  return false                                                                      # Trả về False nếu check toàn bộ bước đi có thể mà vẫn chưa tìm được lời giải
end procedure

function SOLVE_KNIGHTS_TOUR(N, start_x, start_y)
  Initiallize N x N board with -1      # Tạo bảng n x n với các điểm chưa di chuyển qua, mark = -1
  board[start_x][start_y] <- 0         # Tại vị trí bắt đầu, cập nhật bước = 0
  if KNIGHTS_TOUR(start_x, start_y, 1, board) then
    Print the board
  else
    Print "No solution exists"
  end if
end function

possible_moves = [
  ( 1,  2), ( 1, -2),
  ( 2,  1), ( 2, -1),
  (-1, -2), (-1,  2),
  (-2, -1), (-2,  1)
]

def knight_tour(x, y, move_count, board):
  N = len(board)
  if move_count == N**2 + 1:
    return True

  for (dx, dy) in possible_moves:
    new_x = x + dx
    new_y = y + dy
    
    if (0 <= new_x < N) and (0 <= new_y < N) and (board[new_x][new_y] == -1):
      board[new_x][new_y] = move_count
    
      if knight_tour(new_x, new_y, move_count+1, board):
        return True
      board[new_x][new_y] = -1
  
  return False
  
def solve_knight_tour(size_of_chessboard, start_x, start_y):
